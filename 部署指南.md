## 🌐 步骤二：部署到 Vercel

### 方法 A: Vercel 界面部署 (最简单)

1. **访问 Vercel**
   - 打开 https://vercel.com
   - 用 GitHub 账号登录

2. **导入项目**
   - 点击 "Add New Project"
   - 选择 "Import Git Repository"
   - 找到 `onchain-casino` 仓库
   - 点击 "Import"

3. **配置项目**
   - Framework Preset: **Next.js** (自动检测)
   - Root Directory: `./` (默认)
   - Build Command: `pnpm build` (或 `npm run build`)
   - Output Directory: `.next` (默认)

4. **添加环境变量**
   点击 "Environment Variables"，添加：
   ```
   OKX_API_KEY=你的 API_KEY
   OKX_SECRET_KEY=你的 SECRET_KEY
   OKX_PASSPHRASE=你的 PASSPHRASE
   ```

5. **部署**
   - 点击 "Deploy"
   - 等待 2-3 分钟
   - 部署成功后会显示预览链接

### 方法 B: Vercel CLI 部署

```bash
# 1. 安装 Vercel CLI
npm install -g vercel

# 2. 登录 Vercel
vercel login

# 3. 部署
cd /home/diaomao/.openclaw/workspace/onchain-casino
vercel

# 4. 部署到生产环境
vercel --prod
```

## 🔧 步骤四：配置 OKX API (生产环境)

## 🐛 常见问题

### Q1: 部署失败 "Build failed"

**解决方法**:
```bash
# 本地测试构建
pnpm build

# 查看错误日志
# 通常是 TypeScript 错误或依赖问题
```

### Q2: API 调用失败 "401 Unauthorized"

**原因**: 环境变量未正确配置

**解决方法**:
1. 检查 Vercel 环境变量是否正确
2. 确认 API 密钥格式正确
3. 重新部署项目

### Q3: 页面显示空白

**原因**: 可能是 CSP 或 CORS 问题

**解决方法**:
1. 检查浏览器控制台错误
2. 确认 `next.config.js` 配置正确
3. 检查 Vercel 部署日志

### Q4: 钱包连接失败

**原因**: 缺少钱包适配器配置

**解决方法**:
1. 检查是否安装了 @solana/wallet-adapter 相关包
2. 确认钱包提供商配置正确
3. 测试多个钱包 (Phantom/MetaMask)

## 📈 性能优化建议

### 1. 启用缓存

在 `vercel.json` 添加:
```json
{
  "headers": [
    {
      "source": "/static/(.*)",
      "headers": [
        {
          "key": "Cache-Control",
          "value": "public, max-age=31536000, immutable"
        }
      ]
    }
  ]
}
```

### 2. 图片优化

使用 Next.js Image 组件:
```tsx
import Image from 'next/image';
<Image src="/logo.png" width={200} height={200} alt="Logo" />
```

### 3. 代码分割

动态导入大组件:
```tsx
import dynamic from 'next/dynamic';
const Chart = dynamic(() => import('./Chart'), { ssr: false });
```
